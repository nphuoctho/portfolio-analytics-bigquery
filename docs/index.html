Loan Portfolio Analytics — BigQuery + dbt

Raw lending data → analytics marts: DPD buckets, vintage curves, rolling collections, and segment health KPIs. SQL window functions on BigQuery, pipeline run by dbt, every number checked against a Python ground truth.

📊 Live dashboard 💻 Source code

Pipeline

raw (seeds)  →  staging (views)        →  marts (tables)
customers   →  stg_customers  ┐
loans       →  stg_loans      ├─→ mart_loan_dpd_snapshot ─→ mart_portfolio_health
repayments  →  stg_repayments ┘   mart_vintage_curve
                                  mart_rolling_collection

One dbt build rebuilds seeds, models, and tests.

Data quality

  • dbt schema tests: not_null, unique, relationships, accepted_values, accepted_range.
  • Point-in-time integrity test: no payment recorded after the as-of date.
  • Business-rule test: DPD ≥ 0, amounts ≥ 0, no overpayment.
  • Python validator recomputes the marts from CSV; the numbers match BigQuery.
